📂 1.1 Folder Automation (ဖိုင်ပေါင်း ၂၆၀ ကျော်ကို တစ်ချက်တည်းပေါင်းခြင်း)

❓ ဘာကြောင့် ဒါကို လေ့လာရသလဲ? (Business Need)

လုပ်ငန်းခွင်မှာ Data တွေကို နေ့စဉ်၊ လအလိုက် ဖိုင်တွေခွဲပြီး သိမ်းလေ့ရှိပါတယ်။ ဥပမာ - နယ်ရုံးခွဲပေါင်း ၂၀ ကျော်က ပို့သမျှ Sales ဖိုင်တွေကို တစ်နှစ်စာပြန်ပေါင်းကြည့်ရင် ဖိုင်ပေါင်း ၂၄၀ ကျော် ရှိနေပါပြီ။ ဒါတွေကို ပေါင်းပြီးမှသာ Trend Analysis တွေ၊ Dashboard တွေ ဆွဲလို့ရမှာမို့လို့ ဖြစ်ပါတယ်။

❌ Manual Excel Way

  • • ဖိုင် ၂၀၀ ကျော်ကို တစ်ခုချင်း ဖွင့်ရတယ်။
  • • Copy ကူးပြီး Master ဖိုင်မှာ Paste လိုက်လုပ်ရတယ်။
  • • ဖိုင်အသစ် ထပ်လာရင် နောက်တစ်ခါ ပြန်ကူးရပြန်တယ်။
  • • လူပင်ပန်းပြီး Human Error (ဒေတာကျန်ခဲ့တာ) ဖြစ်နိုင်တယ်။

✅ Power Query "Robotic Mode"

  • • Folder လမ်းကြောင်း တစ်ခါပဲ ချိတ်ပေးဖို့ လိုတယ်။
  • • ဖိုင်အသစ် ၁၀ ဖိုင် ထပ်ရောက်လာလည်း ဘာမှ ထပ်လုပ်စရာမလို။
  • • "Refresh" နှိပ်လိုက်တာနဲ့ စက္ကန့်ပိုင်းအတွင်း အကုန်ပေါင်းပြီးသား။
  • • Flexible ဖြစ်တယ် (ဖိုင်တွေ တိုးလာလည်း Error မတက်ဘူး)။

💡 ဒီသင်ခန်းစာရဲ့ အဓိက လက်နက် - "Excel Tables"

ဒီမှာ ကျွန်တော်တို့က Worksheet တွေကို မသုံးဘဲ Excel Tables တွေကိုပဲ သုံးပါမယ်။ ဘာလို့လဲဆိုတော့ Table တွေက Dynamic ဖြစ်လို့ ဒေတာ အတိုးအလျှော့ ရှိလာရင်တောင် Query က အတိအကျ သိနေမှာ ဖြစ်လို့ပါ။

📄 2.1 Mastering CSV Data Imports (System Data Cleaning)

❓ ဘာကြောင့် CSV Import ကို သီးသန့်လေ့လာရသလဲ?

လုပ်ငန်းသုံး System အများစု (ERP, SAP, Web Systems) ကနေ ဒေတာဆွဲချလိုက်ရင် CSV (Comma-Separated Values) Format နဲ့ ထွက်လာလေ့ရှိပါတယ်။ ဒီဖိုင်တွေမှာ အပေါ်ဆုံးက ခေါင်းစဉ်ပိုတွေ၊ အောက်ခြေက "Page 1 of 1" ဆိုတဲ့ Footer တွေ ရောနှောပါဝင်နေတတ်လို့ သာမန် Excel နဲ့ ရှင်းရတာ အချိန်ကုန်စေပါတယ်။

💡 ဒီသင်ခန်းစာရဲ့ Key Techniques:

  • Dynamic Filtering: Row အရေအတွက် ဘယ်လောက်ပြောင်းပြောင်း အမှိုက်တွေကို အလိုအလျောက် သိရှိရှင်းလင်းပေးမယ့် နည်းလမ်း။
  • Using Locale: Date Format Error ကင်းစင်အောင် နိုင်ငံတကာစံနှုန်း (Region) အလိုက် ဒေတာအမျိုးအစား သတ်မှတ်နည်း။

"တစ်ခါ Setup လုပ်ပြီးတာနဲ့ System ကနေ နောက်တစ်ခါ CSV အသစ်ဆွဲချတိုင်း Copy-Paste လုပ်စရာမလိုဘဲ Refresh နှိပ်ရုံနဲ့ ဒေတာတွေ Update ဖြစ်နေမှာပါ။"

📂 2.2 Masterpiece: Multiple CSV Consolidation

🏦 Banking & Enterprise Power

Core Banking System သို့မဟုတ် ERP များမှ နေ့စဉ် Export ထုတ်ယူရသော ဒေတာများသည် များသောအားဖြင့် CSV format များဖြစ်ကြပါသည်။ ဖိုင်အရေအတွက် ရာနှင့်ချီရှိနေစေကာမူ Manual တစ်ခုချင်း စုစည်းနေစရာမလိုဘဲ Query တစ်ခုတည်းအတွင်းသို့ စီးဝင်စေမည့် နည်းပညာဖြစ်ပါသည်။

⚡
Zero Copy-Paste

ဖိုင်ဘယ်လောက်များများ တစ်ခါတည်းပေါင်းပေးတယ်။

📈
Scale-ability

ဒေတာပမာဏ သန်းချီရှိလာလည်း အမြဲမြန်ဆန်နေမည်။

🤖
Robotic Update

ဖိုင်သစ်ရောက်လာရင် Refresh နှိပ်ရုံဖြင့် ပြီးစီးမည်။

✨ Why It’s Smart?

တစ်နေ့ကို ၁ နာရီကြာအောင် ဖိုင်ပေါင်းနေရတဲ့ အလုပ်ကို ၅ စက္ကန့်တည်းနဲ့ ပြီးအောင် Automation လုပ်နိုင်သွားမှာပါ။ ဤနည်းပညာကို တတ်မြောက်ထားခြင်းဖြင့် ရိုးရိုး Excel သမားထက် သာလွန်သည့် "Automation Expert" တစ်ဦးအဖြစ် လုပ်ငန်းခွင်မှာ ရပ်တည်နိုင်မည် ဖြစ်ပါသည်။

📊 2.3 Single Excel Sheet Import (Filter by Kind)

❓ CSV နဲ့ Excel Import ဘာကွာသလဲ?

CSV က ရိုးရှင်းတဲ့ စာသားဖိုင်ဖြစ်ပေမယ့် Excel ဖိုင်တွေထဲမှာတော့ Worksheet တွေအပြင်၊ Tables တွေနဲ့ Defined Names (Range) တွေ အများကြီး ရောထွေးနေတတ်ပါတယ်။ ဒါကြောင့် Power Query နဲ့ ခေါ်တဲ့အခါ ဒေတာတွေ ထပ်မသွားအောင် စနစ်တကျ ရွေးချယ်တတ်ဖို့ လိုအပ်ပါတယ်။

🎯
Precision Select

လိုချင်တဲ့ Sheet ကိုပဲ အတိအကျ ရွေးထုတ်နိုင်ခြင်း။

🛡️
Zero Redundancy

ဒေတာတွေ နှစ်ခါထပ်မလာအောင် "Kind" နဲ့ စစ်ထုတ်ခြင်း။

⚡
Auto Shaping

ခေါင်းစဉ်အပိုတွေကို Dynamic ဖယ်ရှားပြီးသား ဖြစ်ခြင်း။

"Manual Copy-Paste လုံးဝမလိုတော့ဘဲ၊ မူလ Excel ဖိုင်မှာ ဒေတာအသစ်တွေ ထပ်တိုးတာနဲ့ Refresh တစ်ချက်နှိပ်ရုံဖြင့် သင့် Dashboard က အလိုအလျောက် အဆင်သင့် ဖြစ်နေပါလိမ့်မယ်။"

🗝️ 2.4 Multiple Sheets Consolidation (The Pro Secret)

📂 Multi-Sheet Management Challenge

လုပ်ငန်းခွင်တွင် Data များကို Excel ဖိုင်တစ်ခုတည်းအတွင်း Sheet များစွာခွဲ၍ (ဥပမာ - Jan, Feb, Mar...) သိမ်းဆည်းလေ့ရှိကြပါသည်။ ဤဒေတာများကို ပေါင်းစပ်ရန်အတွက် Blank Query ခေါ်ပြီး Code ရေးစရာမလိုဘဲ ကျွမ်းကျင်သူများသာ အသုံးပြုလေ့ရှိသည့် "Step Stripping" နည်းပညာဖြင့် အလိုအလျောက် ပေါင်းစည်းနည်းကို လေ့လာရမည်ဖြစ်ပါသည်။

✂️
Step Stripping

မလိုအပ်သော Steps များကို ဖြတ်ထုတ်ကာ Flow ကို ပေါ့ပါးစေခြင်း။

🧱
Data Stacking

Sheet ပေါင်းများစွာရှိ ဒေတာများကို အပေါ်အောက် အလိုအလျောက် စုစည်းခြင်း။

🛡️
Dynamic Clean

ခေါင်းစဉ်အပိုများနှင့် အမှိုက်များကို တစ်ချက်တည်း ရှင်းလင်းခြင်း။

✨ Why It’s a Pro Secret?

ရိုးရိုး Navigator မှတစ်ဆင့် ခေါ်ယူခြင်းသည် Sheet တစ်ခုတည်းကိုသာ ရည်ညွှန်း (Hardcode) ထားခြင်း ဖြစ်သဖြင့် နောင်တွင် Sheet အသစ်များ တိုးလာပါက အလိုအလျောက် ပါဝင်လာမည် မဟုတ်ပါ။ ဤ Pro Secret ကို အသုံးပြုခြင်းဖြင့် နောင်တစ်ချိန် Excel ဖိုင်အတွင်း Sheet အသစ်များ မည်မျှပင် ထပ်တိုးလာစေကာမူ "Refresh" တစ်ချက်နှိပ်ရုံဖြင့် အကုန်လုံး ပေါင်းစည်းပြီးသား ဖြစ်နေပါမည်။

🏆 3.1 The Ultimate Masterpiece: Files & Sheets Consolidation

📂 Manual Excel User များ၏ နေ့စဉ် Pain Points များ

❌ Manual Excel User တို့၏ အခက်အခဲများ
  • • Double Work: ဖိုင်အများကြီးကို တစ်ခုချင်းဖွင့်ရသလို၊ ထဲမှာရှိတဲ့ Sheet တွေကိုပါ တစ်ခုချင်း Copy လိုက်ကူးနေရခြင်း။
  • • Header Hell: ပေါင်းလိုက်တဲ့အခါ ကြားထဲမှာ ညှပ်ပါလာတဲ့ ခေါင်းစဉ် (Header) တွေကို Manual လိုက်ဖျက်နေရခြင်း။
  • • High Risk: ဒေတာတစ်ခုခု ကျန်ခဲ့ရင် ရှာရခက်ပြီး Human Error ကြောင့် Report မှားယွင်းနိုင်ခြင်း။
✅ The Robotic Power Query Way
  • • One-Click Magic: ဖိုင်ဘယ်လောက်များများ၊ Sheet ဘယ်လောက်ရှိရှိ Refresh တစ်ချက်နှိပ်ရုံဖြင့် အကုန်ပေါင်းပေးခြင်း။
  • • Dynamic Cleaning: ထပ်နေတဲ့ ခေါင်းစဉ် (Headers) တွေနဲ့ အမှိုက် (Nulls) တွေကို စက္ကန့်ပိုင်းအတွင်း အမြစ်ပြတ်ရှင်းလင်းခြင်း။
  • • Future Proof: နောက်လမှာ ဖိုင်အသစ် သို့မဟုတ် Sheet အသစ် တိုးလာလည်း ဘာမှ ထပ်ပြင်စရာမလိုဘဲ အဆင်သင့်ဖြစ်နေခြင်း။

🏆 သင်ယူရမည့် ရလဒ်:

နာရီပေါင်းများစွာ ကြာမြင့်မည့် ရှုပ်ထွေးလှသော Consolidation အလုပ်များကို စက်ရုပ်တစ်ရုပ်ကဲ့သို့ တိကျမြန်ဆန်စွာ Automate လုပ်နိုင်သွားပါမည်။

🌐 3.2 External Sources (Web Scraping & SQL Database)

🚀 Beyond Excel Files

ခေတ်မီလုပ်ငန်းခွင်များတွင် အချက်အလက်များသည် Excel သို့မဟုတ် CSV များတွင်သာ ရှိနေသည်မဟုတ်ပါ။ Wikipedia ကဲ့သို့ Website များမှ နေ့စဉ်ပြောင်းလဲနေသော Live Data များကို ဆွဲယူရန် လိုအပ်သလို၊ လုပ်ငန်း၏ ပင်မ Database (SQL Server) ဆီသို့ တိုက်ရိုက်ချိတ်ဆက်၍ ခိုင်မာသော Data Architecture တစ်ခု တည်ဆောက်တတ်ရန်လည်း အရေးကြီးပါသည်။

🌍
Web Scraping

Website များပေါ်ရှိ Live Tables များကို တိုက်ရိုက်ဆွဲယူခြင်း။

🗄️
SQL Integration

Company Database နှင့် တိုက်ရိုက် Bridge တည်ဆောက်ခြင်း။

🔄
Auto Live Update

ရင်းမြစ်တွင် ဒေတာပြောင်းသည်နှင့် Report ပါ ချက်ချင်းပြောင်းခြင်း။

✨ The Death of "Export to Excel"

စနစ်တစ်ခုမှ Excel ကို နေ့စဉ် Download ထုတ်ပြီး Report ပြန်ဆွဲနေရသည့် ခေတ်ကုန်သွားပါပြီ။ ဤသင်ခန်းစာတွင် Direct DB Connection နည်းပညာကို အသုံးပြု၍ လူသားအမှား (Human Error) များကို ၁၀၀% ကာကွယ်နည်းနှင့် File Size အကန့်အသတ်မရှိ (Big Data) ကို ကိုင်တွယ်နည်းများကို လေ့လာရမည်ဖြစ်ပါသည်။

⏬ 3.3 Fill Data & Shaping (Excel Merge & Center ပြဿနာကို ရှင်းခြင်း)

❓ "Merge & Center" ကြောင့် ဒေတာတွေ ပျောက်ဆုံးနေပါသလား?

လုပ်ငန်းခွင်သုံး Excel Report အများစုတွင် ကြည့်ရှုရ လွယ်ကူစေရန် အချက်အလက်များကို အထက်အောက် Merge လုပ်၍ တည်ဆောက်လေ့ရှိပါသည်။ သို့သော် Database အမြင်တွင် ထိုကွက်လပ်များသည် "Null" (ဘာမှမရှိသောနေရာ) များဖြစ်သွားပြီး Data Analysis လုပ်ရာတွင် ကြီးမားသော အဟန့်အတား ဖြစ်စေပါသည်။

🪄
Fill Down Magic

အပေါ်ကဒေတာကို အောက်ကကွက်လပ်တွေထဲ Auto ဖြည့်ပေးခြင်း။

📋
Tabular Structure

ဖရိုဖရဲ Report မှ သန့်ရှင်းသော Database ဇယားသို့ ပြောင်းလဲခြင်း။

🚀
Instant Analysis

ဒေတာပြည့်စုံသွားသဖြင့် Dashboard ဆွဲရန် အသင့်ဖြစ်ခြင်း။

💡 Data Engineer တိုင်း သိထားရမည့် Core Skill

ကွက်လပ် (null) များတွင် ဒေတာများကို Manual လိုက်လံရိုက်ထည့်ခြင်း သို့မဟုတ် Copy-Paste လုပ်ခြင်းသည် ဒေတာပမာဏ များလာပါက မဖြစ်နိုင်တော့ပါ။ Fill Down နည်းပညာကို အသုံးပြုခြင်းဖြင့် စာကြောင်းရေ သောင်းနှင့်ချီသော Report များကို စက္ကန့်ပိုင်းအတွင်း သန့်စင်ပြီးသား "Ready-to-Structure Data" အဖြစ် ပြောင်းလဲနိုင်မည် ဖြစ်ပါသည်။

[Image of a spreadsheet before and after applying the Fill Down transformation in Power Query]

🏆 Result: Transform "Human-Readable" Reports into "Machine-Ready" Data

⚙️ 3.4 Advanced Shaping: Bulk Fill Down Automation

📂 The Multiple Merged-File Challenge

ယခင်သင်ခန်းစာတွင် ဖိုင်တစ်ဖိုင်တည်း၏ ပြဿနာကို ဖြေရှင်းခဲ့သော်လည်း၊ လက်တွေ့တွင်မူ ဖိုင်ပေါင်းများစွာသည် Merged Cells များဖြင့် Folder အတွင်း ပြန့်ကျဲနေတတ်ပါသည်။ ဤဖိုင်များအားလုံးကို တစ်ပြိုင်နက်တည်း ပေါင်းစပ်ပြီး "Null" ပြဿနာများကို တစ်ချက်တည်းဖြင့် အမြစ်ပြတ်ရှင်းလင်းမည့် Automation Pipeline တစ်ခုကို တည်ဆောက်ရမည် ဖြစ်ပါသည်။

📦
Bulk Extraction

ဖိုင်များစွာမှ Merged Data များကို တစ်ပြိုင်နက် ဆွဲထုတ်ခြင်း။

🛡️
Header Trap Fix

ဒေတာများကြား ညှပ်ပါလာသော ခေါင်းစဉ်အပိုများကို ဖယ်ရှားခြင်း။

⏬
Global Fill Down

ဖိုင်အားလုံးရှိ ကွက်လပ်များကို စက္ကန့်ပိုင်းအတွင်း ဖြည့်တင်းခြင်း။

💡 အရေးအကြီးဆုံးအချက်: The Duplicate Header Trap

ဖိုင်များစွာကို ပေါင်းစပ်လိုက်သောအခါ ဒုတိယနှင့် တတိယဖိုင်များမှ "Column Headers" များသည် ဒေတာများအလယ်တွင် စာသားများအဖြစ် ပိတ်မိနေတတ်ပါသည်။ ၎င်းတို့ကို Filter ဖြင့် မဖယ်ရှားပါက Data Type ပြောင်းသည့်အခါ Error တက်မည် ဖြစ်ပါသည်။ ဤသည်မှာ Folder Automation ၏ အသက်သွေးကြော ဖြစ်ပါသည်။

🏆 Ultimate Goal:

အလုပ်ချိန် နာရီပေါင်းများစွာကို သက်သာစေပြီး၊ ၁၀၀% တိကျမှန်ကန်သော Automated Master Database တစ်ခုကို တည်ဆောက်နိုင်သွားမည်။

🔄 3.5 Transpose Magic: Fixing Horizontal Merged Cells

❓ အလျားလိုက် Merge လုပ်ထားသော ခေါင်းစဉ်များ

Excel တွင် Report များကို ဖတ်ရလွယ်ကူစေရန် ခေါင်းစဉ်များကို အလျားလိုက် (Horizontally) Merge လုပ်ထားလေ့ရှိပါသည်။ သို့သော် Power Query တွင် "Fill Right" ဆိုသည့် Function မရှိသဖြင့် အဆိုပါ ဘေးတိုက်ကွက်လပ်များကို ဖြည့်တင်းရန်မှာ အလွန်ခက်ခဲလှပါသည်။

🔄
Transpose Move

ဇယားကို ၉၀ ဒီဂရီ လှည့်၍ ဘေးတိုက်ကွက်လပ်ကို ဒေါင်လိုက်ပြောင်းခြင်း။

🔗
Unique Header

Main Category နှင့် Sub-Category ကို တစ်ပေါင်းတည်း ဖြစ်စေခြင်း။

🔁
Transpose Back

သန့်စင်ပြီးနောက် ဇယားကို မူလအနေအထားသို့ ပြန်လှည့်ခြင်း။

💡 The "Ninja" Technique: Double Transpose

ဤနည်းစနစ်သည် ရှုပ်ထွေးလှသော ခေါင်းစဉ်များကို စက်ဖတ်၍ရသော (Machine-friendly) Database Headers များအဖြစ် ပြောင်းလဲရာတွင် အထိရောက်ဆုံး ဖြစ်ပါသည်။ Transpose >> Fill Down >> Merge >> Transpose Back ဆိုသည့် အဆင့်ဆင့်သော logic ကို အသုံးပြုခြင်းဖြင့် Manual တစ်ချက်မှ ရိုက်စရာမလိုဘဲ အားလုံးကို Automate လုပ်နိုင်သွားပါမည်။

"Data Shaping is not just about tools, it's about the LOGIC you apply."

🔄 3.6 Transpose Master: Bulk Merged-Cell Automation

📂 The "Master Level" Challenge

ယခင်သင်ခန်းစာတွင် ဖိုင်တစ်ဖိုင်တည်း၏ Horizontal Merge ပြဿနာကို ဖြေရှင်းခဲ့သော်လည်း၊ အလုပ်ခွင်တွင် ထိုကဲ့သို့ ဖရိုဖရဲ Merge လုပ်ထားသော ဖိုင်ပေါင်းရာချီကို Folder လိုက် တစ်ပြိုင်နက်တည်း Automate လုပ်ရန် လိုအပ်လာပါသည်။ ဤသည်မှာ သာမန် Excel User တစ်ဦးအတွက် မဖြစ်နိုင်သလောက် ခက်ခဲသော အလုပ်ဖြစ်ပါသည်။

🔄
Bulk Transpose

ဖိုင်များစွာ၏ ဇယားများကို တစ်ပြိုင်နက်လှည့်၍ သန့်စင်ခြင်း။

⏬
Fill & Merge

Category အားလုံးကို အလိုအလျောက် ဖြည့်တင်းပေါင်းစပ်ခြင်း။

🛡️
Header Trap Fix

ဖိုင်အသီးသီးမှ ညပ်ပါလာသော ခေါင်းစဉ်အပိုများကို ရှင်းထုတ်ခြင်း။

💡 The Strategy: Robotic Transformation Pipeline

ဤသင်ခန်းစာတွင် Transpose > Fill Down > Merge > Transpose Back ဆိုသည့် "Ninja Strategy" ကို Folder Automation အတွင်းသို့ ထည့်သွင်းအသုံးပြုပါမည်။ ဤနည်းလမ်းသည် ဖိုဒါအတွင်းသို့ ဖိုင်အသစ်များ မည်မျှပင် ထပ်ရောက်လာစေကာမူ Refresh တစ်ချက်နှိပ်ရုံဖြင့် အချက်အလက်အားလုံးကို အလိုအလျောက် ပေါင်းစပ်ပေးသွားမည် ဖြစ်ပါသည်။

🏆 Master Level Outcome:

အလုပ်ချိန် ရက်သတ္တပတ်ပေါင်းများစွာ ကြာမြင့်မည့် Data Cleaning အလုပ်များကို Robotic Mode ဖြင့် တစ်သက်စာအတွက် Automation လုပ်နိုင်သွားမည်။

📅 4.1 Bank Statement Foundation (Messy Data များကို သန့်စင်ခြင်း)

❓ ဘဏ်မှတ်တမ်းများကို Analysis လုပ်ရန် အခက်တွေ့နေပါသလား?

ဘဏ်မှ တိုက်ရိုက်ရရှိသော Excel မှတ်တမ်းများသည် Analysis လုပ်ရန် အဆင်သင့်မဖြစ်သေးသော "Messy Data" များဖြစ်ပါသည်။ အပေါ်ဆုံးတွင် မလိုအပ်သော ဘဏ်အမည်များ ပါဝင်နေတတ်သလို၊ ရက်စွဲများမှာလည်း အချိန် (Time) များနှင့် ရောထွေးနေတတ်ပါသည်။

✂️
Text Extraction

ရှည်လျားသော Description များမှ Transaction Type ကို ခွဲထုတ်ခြင်း။

📅
The Date Fix

Date Error ကင်းစင်အောင် ပုံစံသစ်ဖြင့် ပြန်လည်တည်ဆောက်ခြင်း။

⚖️
Logic-Based Value

Credit/Debit နှစ်ခုကို Value တစ်ခုတည်းအဖြစ် စုစည်းခြင်း။

💡 The "Ultimate Date Fix" Strategy

ဘဏ်စာရင်းများတွင် ရက်စွဲနှင့် အချိန် ရောထွေးနေမှုကြောင့် Date Type ပြောင်းလိုက်လျှင် Error တက်လေ့ရှိပါသည်။ ဤသင်ခန်းစာတွင် ရက်စွဲများကို အပိုင်းပိုင်းဖြတ်တောက် (Split) ပြီး မိမိကွန်ပျူတာ နားလည်သော အစီအစဉ်အတိုင်း ပြန်လည်တပ်ဆင် (Merge) နည်းကို အသုံးပြု၍ ၁၀၀% တိကျသော Data Foundation ကို တည်ဆောက်ပြသွားမည် ဖြစ်ပါသည်။

"Data Foundation ခိုင်မာမှသာ မှန်ကန်သော Dashboard Analysis များကို ဖန်တီးနိုင်မည် ဖြစ်ပါသည်။"

🧮 4.2 Pivot & Unpivot Logic (KPI Achievement တွက်ချက်ခြင်း)

❓ Unpivot လုပ်ထားသော ဒေတာတွင် KPI ကို မည်သို့တွက်မည်နည်း?

ရှုပ်ထွေးသော ဒေတာများကို သန့်စင်ရန် Unpivot သည် အကောင်းဆုံးဖြစ်သော်လည်း၊ Actual နှင့် Target ကို နှိုင်းယှဉ်၍ လုပ်ဆောင်ရည် (KPI) ရာခိုင်နှုန်းတွက်ချက်လိုသောအခါ Unpivot ထားသော ဒေါင်လိုက်ဇယားသည် အခက်အခဲ ဖြစ်စေပါသည်။ ထိုအခါ ဇယားကို လိုသလို ပြန်လည်စုစည်းပေးမည့် Pivot နည်းပညာကို အသုံးပြုရပါမည်။

🧱
Flat Database

Unpivot ဖြင့် ဖရိုဖရဲ ဒေတာများကို ရှင်းလင်းသော Database ပြုလုပ်ခြင်း။

🎯
Pivot for Compare

Target နှင့် Actual ကို ဘေးတိုက်ယှဉ်၍ တွက်ချက်ရန် ပြင်ဆင်ခြင်း။

📈
Auto KPI

Achievement % ကို Flow အတွင်း၌ပင် တစ်ခါတည်း တွက်ချက်ခြင်း။

💡 The Strategy: "Reshaping for Analytics"

ဤသင်ခန်းစာတွင် Unpivot > Group By > Pivot ဆိုသည့် Advance Logic ကို အသုံးပြုသွားမည်ဖြစ်ပါသည်။ ဤနည်းလမ်းသည် အရောင်းဝန်ထမ်းရာပေါင်းများစွာ၏ KPI များကို Manual Excel တွက်ချက်စရာမလိုဘဲ ဖိုင်အသစ်ရောက်လာတိုင်း Refresh တစ်ချက်ဖြင့် အလိုအလျောက် တွက်ချက်ပေးသွားမည်ဖြစ်ပါသည်။

🏆 Result: From Messy Reports to Automated Performance Tracking

🔗 4.3 The Append Magic (Yearly Data Consolidation)

📂 လမ်းကြောင်းမတူသော ဖိုင်များကို ပေါင်းစပ်ခြင်း

အချက်အလက်များကို ဖိုဒါတစ်ခုတည်းတွင် စုစည်းထားခြင်းမရှိဘဲ လမ်းကြောင်း (Path) အမျိုးမျိုးတွင် ပြန့်ကျဲနေသောအခါ Folder Automation ပြုလုပ်၍ မရနိုင်ပါ။ ထိုအခါ သီးခြားစီဖြစ်နေသော Query များကို Lego တုံးများကဲ့သို့ အထက်အောက် တစ်ဆက်တည်း ပေါင်းစပ်ပေးမည့် "Append Queries" နည်းပညာကို အသုံးပြုရပါမည်။

🧱
Append Logic

မတူညီသော Source များကို အထက်အောက် တစ်သားတည်း ပေါင်းစပ်ခြင်း။

⚡
Zero Redundancy

Data Type နှင့် ခေါင်းစဉ်များ 100% ကိုက်ညီအောင် ညှိယူခြင်း။

🚀
Memory Opti

စက်မလေးစေရန် Enable Load ကို စနစ်တကျ စီမံခန့်ခွဲခြင်း။

💡 Professional Secret: Enable Load Management

မူရင်းဖိုင် (Source Queries) များကိုပါ Excel ထဲသို့ တစ်ပြိုင်နက် Load လုပ်နေခြင်းသည် ဖိုင် Size ကို အလဟဿ ကြီးမားစေပြီး စက်ကို လေးလံစေပါသည်။ Enable Load ကို ပိတ်ထားပြီး နောက်ဆုံးရလဒ် (Combined Result) တစ်ခုတည်းကိုသာ Load လုပ်ခြင်းသည် "Professional Data Engineer" များသာ အသုံးပြုလေ့ရှိသည့် အဆင့်မြင့် လုပ်ထုံးလုပ်နည်းဖြစ်ပါသည်။

🏆 Result: Scalable & Optimized Data Pipeline

လမ်းကြောင်းပေါင်းစုံမှ ဒေတာများကို ပေါင်းစပ်ပြီး စွမ်းဆောင်ရည် အမြင့်မားဆုံးသော Reporting System တစ်ခုကို တည်ဆောက်နိုင်သွားမည်။

⬅️ 5.1 Advanced Joins: Customer Segmentation Analysis

❓ ဘယ်သူတွေက ဝယ်သူအသစ်လဲ? ဘယ်သူတွေ ပျောက်သွားပြီလဲ?

ကုမ္ပဏီ၏ Customer Database နှင့် လစဉ် Sales Data များကို တိုက်စစ်ရန် VLOOKUP ထက် သာလွန်သည့် Merge Queries (Joins) နည်းပညာကို အသုံးပြုရပါမည်။ ဤသင်ခန်းစာတွင် Join Kind ၆ မျိုးကို အသုံးပြု၍ ဖောက်သည်များ၏ အပြုအမူကို အသေးစိတ် ခွဲခြားစိတ်ဖြာနည်းကို လေ့လာရမည် ဖြစ်ပါသည်။

⬅️ Left Outer Join

ဖောက်သည်ဟောင်းများထဲမှ မည်သူတွေ ဝယ်နေဆဲ (Active) လဲဆိုတာ စစ်ဆေးခြင်း။

➡️ Right Outer Join

အရောင်းစာရင်းထဲမှ မည်သူတွေက ဖောက်သည်အသစ် (New Customers) လဲဆိုတာ ရှာဖွေခြင်း။

🔄 Full Outer Join

အဟောင်းရော အသစ်ပါ ဒေတာအားလုံးကို ပေါင်းစပ်၍ Master Report တည်ဆောက်ခြင်း။

🎯 Inner Join

နှစ်ဖက်စလုံးတွင် Match ဖြစ်သော Loyalty Customer များကိုသာ သီးသန့်စစ်ထုတ်ခြင်း။

🚫 Anti Joins

Match မဖြစ်သော (ဥပမာ- လုံးဝ လာမဝယ်တော့သူများ) ကိုသာ သီးသန့် ဆွဲထုတ်ခြင်း။

💡 The Power of Join Logic

Join Logic များကို တတ်မြောက်ထားခြင်းဖြင့် ရှုပ်ထွေးလှသော Data တိုက်စစ်ခြင်းများကို Manual Filter လုပ်စရာမလိုဘဲ Conditional Column နှင့် တွဲဖက်ကာ အလိုအလျောက် ခွဲခြားပေးနိုင်သည့် "Smart Pipeline" တစ်ခုကို တည်ဆောက်နိုင်မည် ဖြစ်ပါသည်။

🏆 Result: Scientific Customer Segmentation

ဒေတာများကို အခြေခံ၍ မှန်ကန်သော Business Decision များ ချမှတ်နိုင်သည့် အဆင့်မြင့် Analyst တစ်ဦး ဖြစ်လာမည်။

🔑 6.1 Master Table Logic (Sales, Product & Promotion)

🏗️ Building the Data Architecture

လုပ်ငန်းခွင် Data Automation တွင် အခြေခံအကျဆုံး ဇယားကြီး (၃) ခုရှိပါသည်။ အရောင်းမှတ်တမ်း (Sales)၊ ပစ္စည်းဈေးနှုန်းများ (Products) နှင့် လအလိုက် ပရိုမိုးရှင်းများ (Promotions) တို့ ဖြစ်ပါသည်။ ယနေ့သင်ခန်းစာတွင် ယင်းဇယားများကို လမ်းကြောင်းတစ်ခုတည်းမှ ခေါ်ယူပြီး Master Table ကြီးတစ်ခုတည်းအဖြစ် ပေါင်းစပ်နည်းကို လေ့လာပါမည်။
